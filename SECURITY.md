# Security Policy

## Reporting a Vulnerability

Please report security vulnerabilities through GitHub Security Advisories for
this repository.

Do not open a public issue for vulnerabilities, leaked secrets, private account
data, or exploit details.

## What to Include

Include enough information for maintainers to reproduce and assess the issue:

- Affected command or file.
- Impact.
- Reproduction steps.
- Expected behavior.
- Any known workaround.

Do not include real credentials, private keys, wallet secrets, access tokens, or
personal account data.

## Supported Versions

Security fixes are prioritized for the latest published npm version.
